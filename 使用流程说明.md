# 艺术课程排课系统 - 教师/学生账号关联流程

## 功能说明

系统现在支持教师和学生先自己注册账号，然后由管理员在后台添加详细信息并关联账号。

---

## 完整使用流程

### 步骤1：教师/学生自己注册账号

1. 访问系统注册页面：http://localhost:5173
2. 点击"注册"按钮
3. 填写注册信息：
   - 用户名（用于登录）
   - 密码
   - 姓名
   - 角色（选择"教师"或"学生"）
4. 点击"注册"完成账号创建

**注意**：此时账号已创建，但还没有关联到教师/学生详细信息，无法查看课表和个人数据。

---

### 步骤2：管理员添加教师/学生信息并关联账号

#### 添加教师信息

1. 使用管理员账号登录系统
   - 用户名：`admin`
   - 密码：`admin123`

2. 进入"教师管理"页面

3. 点击"添加教师"按钮

4. 填写教师详细信息：
   - 姓名
   - 联系电话
   - 专业
   - 可授课程
   - 入职时间
   - **关联账号**：从下拉列表中选择刚才注册的教师账号

5. 点击"确定"保存

**关联账号下拉列表说明**：
- 只显示"未关联"的教师账号
- 显示格式：`姓名 (用户名)`
- 如果列表为空，说明所有教师账号都已关联

#### 添加学生信息

流程与添加教师类似：

1. 进入"学生管理"页面
2. 点击"添加学生"
3. 填写学生详细信息并选择关联账号
4. 保存

---

### 步骤3：教师/学生登录使用系统

关联完成后，教师/学生可以：

1. 使用注册的账号登录系统
2. 查看个人课表
3. 查看个人信息
4. 教师可以提交调课申请
5. 学生可以查看自己的课程安排

## 技术实现说明

### 后端改动

1. **TeacherRequest.java** 和 **StudentRequest.java**
   - 添加了 `userId` 字段，用于接收要关联的用户ID

2. **TeacherService.java** 和 **StudentService.java**
   - 在 `create()` 和 `update()` 方法中，如果提供了 `userId`，会自动更新用户表的 `teacher_id` 或 `student_id` 字段

3. **AuthController.java**
   - 新增 `/api/auth/unlinked-users` 接口
   - 参数：`role`（teacher 或 student）
   - 返回：未关联的用户列表

### 前端改动

1. **教师管理页面（src/views/teacher/List.vue）**
   - 表单中添加"关联账号"下拉选择框
   - 打开表单时自动加载未关联的教师账号列表
   - 提交时将 `userId` 发送到后端

2. **学生管理页面**
   - 同样的改动（需要类似修改）

## 常见问题

### Q1: 为什么关联账号列表是空的？

**答**：所有该角色的账号都已经关联了。需要先让教师/学生注册新账号。

### Q2: 注册后为什么看不到课表？

**答**：注册只创建了登录账号，还需要管理员在后台添加详细信息并关联账号。

### Q3: 可以一个账号关联多个教师/学生吗？

**答**：不可以。一个账号只能关联一个教师或一个学生。

### Q4: 已经关联的账号可以重新关联吗？

**答**：可以。在编辑教师/学生信息时，可以重新选择关联账号。

## 数据库结构

### users 表
```sql
- id: 用户ID
- username: 登录用户名
- password: 密码（BCrypt加密）
- name: 姓名
- role: 角色（admin/teacher/student）
- teacher_id: 关联的教师ID（如果是教师）
- student_id: 关联的学生ID（如果是学生）
```

### teachers 表
```sql
- id: 教师ID
- name: 姓名
- phone: 电话
- specialty: 专业
- join_date: 入职日期
```

### students 表
```sql
- id: 学生ID
- name: 姓名
- age: 年龄
- gender: 性别
- parent_name: 家长姓名
- phone: 联系电话
- enroll_date: 入学日期
```
