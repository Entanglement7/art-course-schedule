# 艺术排课系统 - 数据库导入说明

## 📋 文件信息

- **文件名**: `art_schedule_complete.sql`
- **数据库名**: `art_schedule`
- **字符集**: `utf8mb4`
- **排序规则**: `utf8mb4_unicode_ci`
- **文件大小**: 约 26KB
- **导出时间**: 2026-05-07

## 🔧 系统要求

- MySQL 5.7 或更高版本
- MySQL 8.0+ 推荐

## 📥 导入方法

### 方法一：命令行导入（推荐）

```bash
# Windows
mysql -u root -p --default-character-set=utf8mb4 < art_schedule_complete.sql

# Linux/Mac
mysql -u root -p --default-character-set=utf8mb4 < art_schedule_complete.sql
```

### 方法二：MySQL客户端导入

1. 登录MySQL：
```bash
mysql -u root -p
```

2. 执行导入：
```sql
SOURCE C:/Users/你的路径/art_schedule_complete.sql;
```

### 方法三：使用图形化工具

**Navicat / MySQL Workbench / phpMyAdmin**

1. 打开工具
2. 选择"运行SQL文件"或"导入"
3. 选择 `art_schedule_complete.sql`
4. 确保字符集设置为 `utf8mb4`
5. 执行导入

## ✅ 验证导入

导入完成后，执行以下命令验证：

```sql
-- 切换到数据库
USE art_schedule;

-- 查看所有表
SHOW TABLES;

-- 查看数据库字符集
SHOW CREATE DATABASE art_schedule;

-- 检查数据
SELECT COUNT(*) FROM users;
SELECT COUNT(*) FROM classes;
SELECT COUNT(*) FROM schedules;
```

## 📊 数据库包含的表

- `users` - 用户表（管理员、教师、学生）
- `teachers` - 教师信息表
- `students` - 学生信息表
- `classrooms` - 教室表
- `classroom_equipment` - 教室设备表
- `courses` - 课程表
- `classes` - 班级表
- `class_students` - 班级学生关联表
- `schedules` - 课表表
- `schedule_students` - 课表学生关联表
- `adjust_requests` - 调课申请表
- `attendance` - 考勤记录表

## ⚠️ 注意事项

1. **字符编码**：必须使用 `utf8mb4` 字符集，否则中文会乱码
2. **覆盖警告**：导入会删除已存在的 `art_schedule` 数据库，请提前备份
3. **权限要求**：需要有创建数据库和表的权限
4. **时区设置**：数据库使用 `Asia/Shanghai` 时区

## 🔄 配置应用连接

导入数据库后，修改后端配置文件：

**backend/src/main/resources/application.yml**

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/art_schedule?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai
    username: root
    password: 你的密码
    driver-class-name: com.mysql.cj.jdbc.Driver
```

## 🐛 常见问题

### 1. 中文乱码
确保导入时指定字符集：
```bash
mysql -u root -p --default-character-set=utf8mb4 < art_schedule_complete.sql
```

### 2. 权限不足
```sql
GRANT ALL PRIVILEGES ON art_schedule.* TO 'root'@'localhost';
FLUSH PRIVILEGES;
```

### 3. 导入失败
检查MySQL版本和错误日志：
```bash
mysql --version
```

## 📞 技术支持

如遇到问题，请检查：
1. MySQL服务是否正常运行
2. 字符集设置是否正确
3. 文件路径是否正确
4. 用户权限是否足够
