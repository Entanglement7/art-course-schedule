# 艺术培训机构排课系统技术框架

## 一、系统架构概述

本系统采用**前后端分离架构**，前端基于 **Vue 3** 框架开发单页应用，后端采用 **Spring Boot 3** 构建 RESTful API 服务，数据库使用 **MySQL 8**。

### 1.1 技术栈

**前端技术栈：**

- **框架**：Vue 3.5（Composition API）
- **语言**：TypeScript
- **UI 组件库**：Naive UI 2.44
- **状态管理**：Pinia 3
- **路由**：Vue Router 5
- **网络请求**：Axios 1.7（封装统一请求拦截器）
- **构建工具**：Vite 8
- **日期处理**：Day.js 1.11

**后端技术栈：**

- **框架**：Spring Boot 3.2.5
- **语言**：Java 17
- **ORM**：MyBatis-Plus 3.5.7（分页插件、逻辑删除）
- **数据库**：MySQL 8.0
- **认证**：Spring Security + JWT（jjwt 0.12.5，HMAC-SHA256 签名）
- **密码加密**：BCrypt
- **构建工具**：Maven（内置 mvnw wrapper）

## 二、前端界面模块

### 2.1 登录页（Login.vue）

**功能描述**：系统统一入口，提交用户名和密码，获取 JWT Token 并存储。

**技术实现**：

- 调用 `POST /api/auth/login` 接口，成功后将 token 存入 `localStorage`
- 用户信息（id、name、role、teacherId、studentId）存入 Pinia Store 并持久化到 `localStorage`
- 根据角色跳转至对应首页

**核心代码逻辑**：

```typescript
const handleLogin = async () => {
  const data = await login({ username: form.username, password: form.password })
  localStorage.setItem('token', data.token)
  userStore.setUserInfo(data.userInfo)
  router.push('/dashboard')
}
```

### 2.2 仪表盘（dashboard/Index.vue）

**功能描述**：登录后首页，按角色展示统计数据和待办事项。

**技术实现**：

- 调用 `GET /api/dashboard`，后端依据当前 JWT 中的角色自动返回对应数据
- 管理员视图：教师总数、学生总数、本周排课数、待审核调课申请数
- 教师视图：本周课时数、所带班级数、待审核申请数
- 学生视图：本周课程数、所在班级数
- 待办事项从 `GET /api/adjustments?status=待审核&size=100` 实时拉取，点击"处理"跳转至调课管理页

### 2.3 课表查看模块（timetable/）

#### 2.3.1 周课表（Week.vue）

**功能描述**：以网格形式展示当前周全部排课，支持切换周次。

**技术实现**：

- 使用 CSS Grid（`grid-template-columns: 100px repeat(7, 1fr)`）构建时间×星期二维网格
- 调用 `GET /api/schedules/week?date=YYYY-MM-DD`，date 传当前周任意日期，后端自动对齐到周一
- 课程卡片按类别着色：音乐（紫）、舞蹈（粉）、美术（橙）
- 点击课程卡片弹出详情 Modal，支持跳转调课申请

**核心代码逻辑**：

```typescript
// 筛选某天某时段的课程
function getCourses(dayOfWeek: number, timeSlot: any) {
  return schedules.value.filter(course => {
    if (course.dayOfWeek !== dayOfWeek) return false
    return (
      (course.startTime >= timeSlot.start && course.startTime < timeSlot.end) ||
      (course.endTime > timeSlot.start && course.endTime <= timeSlot.end) ||
      (course.startTime <= timeSlot.start && course.endTime >= timeSlot.end)
    )
  })
}
```

#### 2.3.2 教师课表（Teacher.vue）

**功能描述**：展示指定教师的周课表，管理员可切换教师，教师角色自动显示本人课表。

**技术实现**：

- 学生角色：使用 `userStore.userInfo.studentId`

#### 2.3.4 教室课表（Classroom.vue）

**功能描述**：展示指定教室的周排课情况。

### 2.4 排课管理模块（schedule/）

#### 2.4.1 手动排课（Manual.vue）

**功能描述**：管理员选择班级、课程、教室、时间后创建排课记录，支持按周重复。

**技术实现**：

- 表单含班级、课程、教室三个级联下拉（均调用各自 options 接口）
- `repeatType` 选"weekly"时显示重复周数输入框
- 调用 `POST /api/schedules/manual`，后端按周数循环生成多条记录

**核心代码逻辑**：

```typescript
const handleSubmit = async () => {
  await createManualSchedule({
    classId: form.classId,
    courseId: form.courseId,
    classroomId: form.classroomId,
    dayOfWeek: form.dayOfWeek,
    startDate: form.startDate,
    startTime: form.startTime,
    endTime: form.endTime,
    repeatType: form.repeatType,
    weekCount: form.weekCount
  })
}
```

#### 2.4.2 调课管理（Adjust.vue）

**功能描述**：教师发起调课申请，管理员审批通过或拒绝。

**技术实现**：

- 列表：`GET /api/adjustments`（教师自动仅显示自己的申请）
- 新建：`POST /api/adjustments`（教师的班级下拉仅加载本人名下班级）
- 审批通过：`PUT /api/adjustments/{id}/approve`（仅管理员）
- 拒绝：`PUT /api/adjustments/{id}/reject` + `{ rejectReason }`（仅管理员）
- 撤销：`DELETE /api/adjustments/{id}`（仅申请人且状态为"待审核"）

### 2.6 网络请求层（utils/request.ts）

**统一 Axios 实例**：

```typescript
const request = axios.create({
  baseURL: 'http://localhost:8080',
  timeout: 10000,
})

// 请求拦截：自动注入 JWT Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一错误处理，401 自动跳转登录
request.interceptors.response.use(
  (res) => {
    if (res.data.code === 200) return res.data.data
    return Promise.reject(new Error(res.data.message || '请求失败'))
  },
  (err) => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      router.push('/login')
    }
    return Promise.reject(err)
  }
)
```

**特性**：

- 自动携带 JWT Token
- 401 响应自动清除 Token 并跳转登录页
- 统一解包 `data.data` 字段，业务层直接拿数据
- 支持 async/await 调用

## 三、后端接口模块

所有接口统一响应格式：

```json
{ "code": 200, "message": "success", "data": { ... } }
```

### 3.1 认证接口（AuthController）

#### 3.1.1 登录接口

- **路由**：`POST /api/auth/login`
- **请求体**：`{ username, password }`
- **业务逻辑**：
  1. 按用户名查询 users 表
  2. BCrypt 验证密码
  3. 生成 JWT Token（有效期 24 小时，包含 userId、username、role 三个 Claim）
  4. 返回 token 和 userInfo

#### 3.1.2 获取当前用户

- **路由**：`GET /api/auth/me`
- **认证**：需要 JWT Token
- **逻辑**：通过 `@AuthenticationPrincipal User user` 直接注入当前登录用户，屏蔽密码字段后返回

**JWT 生成核心代码**：

```java
public String generateToken(Long userId, String username, String role) {
    return Jwts.builder()
            .subject(String.valueOf(userId))
            .claim("username", username)
            .claim("role", role)
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + expiration))
            .signWith(Keys.hmacShaKeyFor(secret.getBytes(StandardCharsets.UTF_8)))
            .compact();
}
```

**手动排课请求体示例**：

  "classroomId": 1,
  "dayOfWeek": 1,

**手动排课逻辑**：

```java
// 从 startDate 起找到第一个符合 dayOfWeek 的日期，按周数循环生成记录
DayOfWeek targetDow = DayOfWeek.of(req.getDayOfWeek());
LocalDate firstDate = startDate.with(TemporalAdjusters.nextOrSame(targetDow));
for (int i = 0; i < weeks; i++) {
    Schedule s = buildSchedule(req, firstDate.plusWeeks(i));

}
```

## 四、数据库设计

### 4.1 核心表结构

```sql
-- 用户表
users(id, username, password, name, role, teacher_id, student_id, created_at)

-- 教师表（逻辑删除）
teachers(id, name, phone, specialty, join_date, created_at, updated_at, deleted)
teacher_courses(teacher_id, course_name)   -- 可授课程，多对多

-- 学生表（逻辑删除）
students(id, name, age, gender, parent_name, phone, enroll_date, created_at, updated_at, deleted)
student_courses(student_id, course_name)   -- 报名课程，多对多

-- 课程表（逻辑删除）
courses(id, name, category, type, duration, age_range, price, description, created_at, updated_at, deleted)

-- 教室表（逻辑删除）
classrooms(id, name, type, capacity, floor, status, remark, created_at, updated_at, deleted)
classroom_equipment(id, classroom_id, equipment_name)   -- 设备，一对多

-- 班级表（逻辑删除）
classes(id, name, category, course_name, teacher_id, student_count, schedule, start_date, created_at, updated_at, deleted)
class_students(class_id, student_id)   -- 学生关联，多对多

-- 排课记录表
schedules(id, class_id, course_id, teacher_id, classroom_id, date, day_of_week, start_time, end_time, status, created_at, updated_at)

-- 调课申请表
adjust_requests(id, applicant_id, class_id, schedule_id, original_time, new_day_of_week, new_start_time, new_end_time, reason, status, reject_reason, apply_time, reviewed_at, reviewer_id)
```

### 4.2 MyBatis-Plus 关键配置

- **逻辑删除**：`deleted` 字段全局配置，查询自动追加 `AND deleted = 0`
- **分页插件**：`MybatisPlusInterceptor` 注入 `PaginationInnerInterceptor`，支持 `Page<T>` 对象分页
- **驼峰映射**：`map-underscore-to-camel-case: true`，数据库下划线字段自动映射 Java 驼峰属性
- **关联查询**：`@TableField(exist=false)` 注解声明非数据库字段（如 `teacherName`、`classroomName`），通过 `@Select` 自定义 JOIN 语句填充

## 五、安全与权限设计

### 5.1 JWT 认证流程

```
客户端                         服务端
  │                              │
  │  POST /api/auth/login        │
  │ ─────────────────────────►  │
  │                              │  1. 验证密码（BCrypt）
  │                              │  2. 生成 JWT（HS256，24h）
  │  { token, userInfo }         │
  │ ◄─────────────────────────  │
  │                              │
  │  GET /api/xxx                │
  │  Authorization: Bearer <jwt> │
  │ ─────────────────────────►  │
  │                              │  JwtAuthFilter 解析 Token
  │                              │  → SecurityContext 注入 User
  │  { code: 200, data: ... }    │
  │ ◄─────────────────────────  │
```

### 5.2 URL 级权限规则（SecurityConfig）

| URL 规则 | 允许角色 |
|----------|----------|
| `/api/auth/login`、`/api/auth/register` | 匿名 |
| `/api/teachers/**`、`/api/students/**`、`/api/courses/**`、`/api/classrooms/**` | ADMIN |
| `GET /api/classes/**` | ADMIN、TEACHER |
| `/api/classes/**`（写操作） | ADMIN |
| `/api/adjustments/*/approve`、`/api/adjustments/*/reject` | ADMIN |
| `/api/adjustments/**` | ADMIN、TEACHER |
| `/api/schedules/manual`、`/api/schedules/auto` | ADMIN |
| `GET /api/schedules/**` | 已登录 |
| `/api/dashboard/**`、`/api/auth/me` | 已登录 |

> 规则按从上到下优先级匹配，更精确的规则必须写在前面。

## 六、系统特色

1. **角色驱动的菜单体系**：前端根据 Pinia 中存储的 `role` 字段动态渲染菜单，管理员、教师、学生看到完全不同的功能入口
2. **双层权限保护**：前端路由守卫拦截未登录访问，后端 Spring Security 过滤链二次鉴权，缺一不可
3. **数据隔离**：教师访问调课申请接口时，后端自动附加 `applicantId = 当前用户ID` 条件，无需前端感知
4. **响应式设计**：基于 `window.innerWidth` 检测移动端，切换侧边栏/抽屉两种导航布局，自动适配手机访问
5. **逻辑删除一致性**：所有基础数据表（教师、学生、课程、教室、班级）统一使用 `deleted` 字段，MyBatis-Plus 全局配置自动过滤已删除记录
