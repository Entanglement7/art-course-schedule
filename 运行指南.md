## 零、先了解：什么是"终端"

本指南中很多步骤需要在"终端"里输入命令。终端就是那个黑色的命令行窗口。

**如何打开终端（cmd）：**

方法一（最简单）：
1. 按住键盘左下角的 `Win` 键，同时按 `R` 键
2. 屏幕中央弹出一个小框，在框里输入 `cmd`
3. 按回车键，黑色窗口就打开了

方法二：
1. 点击桌面左下角的"开始"按钮（Windows 图标）
2. 直接输入 `cmd`
3. 点击"命令提示符"即可打开

**如何在终端里输入命令：**
- 看到 `>` 符号后面就可以输入
- 把命令**完整复制**，在黑色窗口里**右键单击**即可粘贴
- 按**回车键**执行命令
- 命令执行完毕后才会出现新的 `>` 符号，等它出现再输入下一条

## 一、安装必备软件

### 第 1 步：安装 Java 17

Java 是运行后端程序的基础环境，必须安装。

1. 打开浏览器，访问：https://www.oracle.com/java/technologies/downloads/#java17
2. 页面中间找到 **Windows** 选项卡，点击它
3. 找到 **x64 Installer** 那一行，点击右边的下载链接，下载 `.exe` 文件（约 160 MB）
4. 下载完成后，**双击**那个 `.exe` 文件
5. 弹出安装界面，一路点 **Next（下一步）**，最后点 **Close** 完成安装

**验证 Java 是否安装成功：**

打开终端（参考上面的方法），输入以下命令并按回车：
```
java -version
```
如果出现类似 `java version "17.0.x"` 的文字，说明安装成功。

> 如果提示"不是内部或外部命令"，尝试关闭终端重新打开，再试一次。

---

### 第 2 步：安装 MySQL 8

MySQL 是数据库软件，用来存储系统里所有的数据。

1. 打开浏览器，访问：https://dev.mysql.com/downloads/installer/
2. 找到较大的那个安装包（文件名含 `community`，约 450 MB），点击 **Download**
3. 页面会跳转到登录提示，找页面下方的 **"No thanks, just start my download"** 点击，无需注册直接下载
4. 下载完成后，**双击** `.msi` 文件
5. 安装类型选择 **Developer Default**，点 Next
6. 一路点 Next，遇到 **Execute** 按钮时点击等待安装完成
7. 进入密码设置界面时，**密码必须填写 `root`**（后续项目要用到这个密码）
8. 继续点 Next，直到 Finish

> 如果你之前已安装过 MySQL 且密码不是 `root`，请参考[常见问题 Q1](#q1-mysql-密码不是-root-怎么办)

---

### 第 3 步：安装 Node.js

Node.js 是运行前端项目的环境。

1. 打开浏览器，访问：https://nodejs.org/
2. 点击页面上绿色的 **LTS** 按钮（旁边标注"Recommended For Most Users"）下载
3. 下载完成后，**双击**安装包
4. 一路点 **Next**，保持所有默认选项不变，最后点 **Finish**

**验证 Node.js 是否安装成功：**

**关闭之前所有的终端窗口，重新打开一个新的终端**，输入：
```
node -v
```
看到 `v20.x.x` 或更高版本，说明安装成功。

---

### 第 4 步：安装 Git

Git 是用来从网络上下载代码的工具。

1. 打开浏览器，访问：https://git-scm.com/download/win
2. 点击 **64-bit Git for Windows Setup** 下载（约 60 MB）
3. 下载完成后，**双击**安装包
4. **所有选项保持默认**，一路点 **Next**，最后点 **Install**，等待安装完成，点 **Finish**

**验证 Git 是否安装成功：**

**重新打开一个新的终端**，输入：
```
git --version
```
看到 `git version 2.x.x` 说明安装成功。

## 二、下载项目代码

这一步把项目代码从网络下载到你的电脑上。

1. 打开终端
2. 依次输入以下两条命令（每输一条按一次回车，等 `>` 出现后再输下一条）：
   ```
   mkdir C:\Projects
   ```
   ```
   cd C:\Projects
   ```
   > 这两条命令的作用是：在 C 盘创建一个叫 `Projects` 的文件夹，然后进入它。
   > 如果提示文件夹已存在，忽略提示继续下一步即可。

3. 输入以下命令下载项目（需要网络，约几十 MB）：
   ```
   git clone https://github.com/Entanglement7/art-course-schedule.git
   ```
   看到一串下载进度后停止，说明下载成功。项目被保存在 `C:\Projects\art-course-schedule` 文件夹里。

## 三、初始化数据库

这一步把项目需要的表结构和初始数据导入 MySQL，**只需要做一次**。

1. 打开终端，输入以下命令登录数据库（按回车后会提示输入密码）：
   ```
   mysql -u root -p
   ```
2. 光标停在 `Enter password:` 后面，输入 `root`，按回车
   > 输入密码时屏幕上不会显示任何字符，这是正常的，直接输完按回车即可

3. 成功登录后，终端会变成 `mysql>` 开头。输入以下命令创建表结构：
   ```
   source C:/Projects/art-course-schedule/backend/src/main/resources/db/schema.sql
   ```
   按回车，看到若干行 `Query OK` 字样说明成功。

4. 再输入以下命令导入初始数据：
   ```
   source C:/Projects/art-course-schedule/backend/src/main/resources/db/data.sql
   ```
   按回车，同样看到 `Query OK` 字样说明成功。

5. 输入 `exit` 按回车，退出数据库。

## 四、启动后端服务

**注意：后端启动后对应的终端窗口不能关闭，整个使用过程中要一直开着。**

1. 打开一个新的终端窗口
2. 输入以下命令进入后端目录：
   ```
   cd C:\Projects\art-course-schedule\backend
   ```
3. 输入以下命令打包后端程序（首次运行会自动从网络下载依赖，需要等待 5～10 分钟）：
   ```
   mvnw.cmd package -DskipTests
   ```
   看到 `BUILD SUCCESS` 字样说明打包完成。

4. 输入以下命令启动后端：
   ```
   java -jar target\art-schedule-0.0.1-SNAPSHOT.jar
   ```
5. 等待几秒钟，看到如下字样说明启动成功：
   ```
   Started ArtScheduleApplication in x.xxx seconds
   ```

> 这个终端窗口**不能关闭**，最小化放在任务栏即可。

## 五、启动前端页面

**注意：同样，前端终端窗口也不能关闭。此时你应该有两个终端窗口在运行。**

1. 再打开一个**新的**终端窗口（保持后端窗口不动）
2. 输入以下命令进入项目根目录：
   ```
   cd C:\Projects\art-course-schedule
   ```
3. 输入以下命令安装前端依赖（**只需要第一次执行**，约 1～2 分钟）：
   ```
   npm install
   ```
   看到 `added xxx packages` 字样说明安装完成。

4. 输入以下命令启动前端：
   ```
   npm run dev
   ```
5. 看到如下输出说明启动成功：
   ```
   VITE v6.x.x  ready in xxx ms

   ➜  Local:   http://localhost:5173/
   ➜  Network: http://192.168.x.x:5173/
   ```

6. 打开浏览器（Chrome、Edge 均可），在地址栏输入：
   ```
   http://localhost:5173
   ```
   按回车，即可看到登录页面。

7. 使用管理员账号登录：
   - 用户名：`admin`
   - 密码：`admin123`

## 六、在手机上访问

手机和电脑必须**连接同一个 WiFi**。

### 第 1 步：查看电脑 IP 地址

打开终端，输入：
```
ipconfig
```
在输出结果里找到 **"以太网适配器"** 或 **"无线局域网适配器 WLAN"** 下面的 **IPv4 地址**，格式类似 `192.168.1.100`，记下这个数字。

也可以直接看第五步前端启动后终端里 `Network:` 后面显示的地址，就是这个 IP。

### 第 2 步：修改前端配置

用**记事本**或任意文本编辑器打开文件：
```
C:\Projects\art-course-schedule\src\utils\request.ts
```

找到这一行：
```
baseURL: 'http://localhost:8080',
```
把 `localhost` 改成你刚才查到的 IP 地址，例如：
```
baseURL: 'http://192.168.1.100:8080',
```
保存文件（`Ctrl + S`）。

### 第 3 步：手机打开系统

在手机浏览器地址栏输入（把 IP 换成你的实际地址）：
```
http://192.168.1.100:5173
```
即可看到登录页面，系统会自动显示为手机版布局。

> **提示**：手机端左上角有菜单按钮，点击展开导航。

## 八、常见问题

### Q1. MySQL 密码不是 root 怎么办？

用记事本打开文件 `C:\Projects\art-course-schedule\backend\src\main\resources\application.yml`，找到：
```yaml
    password: root
```
把 `root` 改成你自己设置的 MySQL 密码，保存后重新启动后端。

---

### Q2. 输入命令后提示"不是内部或外部命令"

这通常是软件安装完后没有重启终端导致的。**关闭所有终端窗口，重新打开一个新的**，再试一次。

---

### Q3. 后端启动报错"端口 8080 被占用"

打开终端，依次输入以下两条命令：
```
netstat -ano | findstr :8080
```
记下输出最后一列的数字（进程 ID），再输入：
```
taskkill /PID 这里填上面的数字 /F
```
然后重新启动后端。

---

### Q4. npm install 非常慢或中途失败

网络原因导致，切换国内镜像后重试。在终端输入：
```
npm config set registry https://registry.npmmirror.com
```
然后重新执行 `npm install`。

---

### Q5. 浏览器打开是空白页，或提示"Network Error"

按顺序检查：
1. 后端终端窗口是否还开着？有没有报错？（应该看到 `Started ArtScheduleApplication`）
2. 数据库是否导入过？（第三步有没有做）
3. 打开 `C:\Projects\art-course-schedule\src\utils\request.ts`，确认 `baseURL` 是 `http://localhost:8080`

---

### Q6. 手机访问打不开

- 确认手机和电脑连的是**同一个 WiFi**（不是手机热点）
- 确认已经按第六步修改了 `request.ts` 里的 IP 地址
- 尝试临时关闭电脑防火墙（控制面板 → Windows Defender 防火墙 → 关闭）再试

---

### Q7. 每次重新使用系统需要做什么？

每次开机使用时，只需要：

1. 打开一个终端，启动后端：
   ```
   cd C:\Projects\art-course-schedule\backend
   java -jar target\art-schedule-0.0.1-SNAPSHOT.jar
   ```
2. 打开另一个终端，启动前端：
   ```
   cd C:\Projects\art-course-schedule
   npm run dev
   ```
3. 打开浏览器访问 `http://localhost:5173`

MySQL 数据库会开机自动启动，无需手动操作。  
`mvnw.cmd package` 和 `npm install` **只需要第一次执行**，之后不用再做。
