# 艺术培训机构排课系统 - 项目结构说明

## 一、项目概览

本项目采用前后端分离架构，前端使用 Vue 3 框架开发单页应用，后端使用 Spring Boot 3 框架提供 RESTful API 服务。

```plain
art-course-schedule/
├── backend/                    # 后端服务（Spring Boot 3 + MySQL）
├── src/                        # 前端源码（Vue 3 + TypeScript）
│   ├── api/                    # 接口函数层
│   ├── components/             # 公共组件
│   ├── router/                 # 路由配置
│   ├── stores/                 # Pinia 状态管理
│   ├── utils/                  # 工具函数
│   ├── views/                  # 页面视图
│   ├── main.ts                 # 应用入口
│   └── App.vue                 # 根组件
├── public/                     # 静态公共资源
├── index.html                  # HTML 模板
└── vite.config.ts              # Vite 构建配置
```

#### 核心文件说明

**SecurityConfig.java**

- **功能**：定义 URL 级权限规则，配置无状态 Session，注册 JWT 过滤器
- **主要内容**：
  - 公开路径：`/api/auth/login`、`/api/auth/register`
  - 管理员专属：教师/学生/课程/教室管理接口，调课审批接口，手动排课接口
  - 教师可读：`GET /api/classes/**`
  - 管理员+教师：调课申请 CRUD
  - 已登录即可：课表查询、仪表盘

**ScheduleMapper.java**

- **功能**：课表相关所有数据库查询
- **主要内容**：
  - `selectWeekSchedule(monday, sunday)`：查询指定周内全部排课，JOIN 关联班级/课程/教师/教室名称
  - `selectTeacherSchedule(teacherId, monday, sunday)`：指定教师的周课表
  - `selectStudentSchedule(studentId, monday, sunday)`：通过 class_students 关联查询学生课表
  - `selectClassroomSchedule(classroomId, monday, sunday)`：指定教室的周课表
  - `selectByIdWithExtra(id)`：按 ID 查单条记录并 JOIN 填充关联名称，用于新建排课后返回完整数据

**AdjustService.java**

- **功能**：调课申请完整业务逻辑
- **主要内容**：
  - 教师提交申请时校验班级必须属于本人
  - 审批通过/拒绝后调用 `fillExtra()` 填充申请人姓名、班级名等展示字段
  - 撤销申请时校验身份和状态（只能撤销自己的待审核申请）
  - `list()` 方法：教师调用时自动附加 `applicantId` 过滤条件，实现数据隔离

#### request.ts

**功能**：统一封装所有后端 API 调用

**核心内容**：

```typescript
const request = axios.create({
  baseURL: 'http://localhost:8080',
  timeout: 10000,
})

// 请求拦截：自动注入 JWT Token
request.interceptors.request.use((config) => {
  const token = localStorage.getItem('token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// 响应拦截：统一解包 data 字段，401 自动跳转登录
request.interceptors.response.use(
  (res) => {
    if (res.data.code === 200) return res.data.data
    return Promise.reject(new Error(res.data.message))
  },
  (err) => {
    if (err.response?.status === 401) {
      localStorage.removeItem('token')
      router.push('/login')
    }
    return Promise.reject(err)
  }
)
```

**特性**：

- 自动携带 JWT Token
- 401 状态码自动清除 Token 并跳转登录页
- 统一解包响应体 `data.data` 字段，业务层直接拿数据
- Promise 封装支持 async/await

### 3.2 数据表结构

#### users（用户表）

```sql
CREATE TABLE users (
    id          BIGINT AUTO_INCREMENT PRIMARY KEY,
    username    VARCHAR(50) NOT NULL UNIQUE,
    password    VARCHAR(100) NOT NULL,     -- BCrypt 加密
    name        VARCHAR(50) NOT NULL,
    role        VARCHAR(20) NOT NULL,      -- admin / teacher / student
    teacher_id  BIGINT NULL,               -- 关联 teachers.id
    student_id  BIGINT NULL,               -- 关联 students.id
    created_at  DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
);
```

#### schedules（排课记录表）

```sql
CREATE TABLE schedules (
    id           BIGINT AUTO_INCREMENT PRIMARY KEY,
    class_id     BIGINT NOT NULL,
    course_id    BIGINT NOT NULL,
    teacher_id   BIGINT NOT NULL,
    classroom_id BIGINT NOT NULL,
    date         DATE NOT NULL,            -- 具体上课日期
    day_of_week  INT NOT NULL,             -- 1=周一，7=周日
    start_time   TIME NOT NULL,
    end_time     TIME NOT NULL,
    status       VARCHAR(20) NOT NULL DEFAULT 'scheduled'
                                           -- scheduled/completed/cancelled/adjusted
);
```

#### adjust_requests（调课申请表）

```sql
CREATE TABLE adjust_requests (
    id              BIGINT AUTO_INCREMENT PRIMARY KEY,
    applicant_id    BIGINT NOT NULL,       -- 申请人用户ID
    class_id        BIGINT NOT NULL,
    schedule_id     BIGINT NULL,
    original_time   VARCHAR(50) NOT NULL,  -- 原上课时间描述
    new_day_of_week INT NULL,
    new_start_time  TIME NULL,
    new_end_time    TIME NULL,
    reason          VARCHAR(500) NOT NULL,
    status          VARCHAR(20) NOT NULL DEFAULT '待审核',
                                           -- 待审核 / 已通过 / 已拒绝
    reject_reason   VARCHAR(500) NULL,
    apply_time      DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
    reviewed_at     DATETIME NULL,
    reviewer_id     BIGINT NULL
);
```

其余表（teachers、students、courses、classrooms、classes 及关联表）均含 `deleted` 逻辑删除字段，由 MyBatis-Plus 全局过滤。

## 六、项目特点总结

1. **前后端分离**：清晰的目录结构，前后端独立运行，通过 REST API 通信
2. **角色驱动设计**：前端菜单、后端权限均由 `role` 字段驱动，三种角色（管理员/教师/学生）体验完全隔离
3. **双层权限保护**：前端路由守卫 + 后端 Spring Security 双重拦截，安全性高
4. **数据隔离**：教师访问调课接口时后端自动附加身份过滤，无需前端传参
5. **响应式布局**：自动检测屏幕宽度，切换侧边栏和抽屉两种导航模式，支持手机浏览器访问
